简化版设计系统 - 使用自定义组件类

✨ 简化前 vs 简化后

❌ 简化前（冗长的类名）

<button className="px-8 py-3 text-sm border-2 border-text-primary
               bg-text-secondary text-white rounded-md
               hover:bg-text-primary hover:-translate-y-0.5
               transition-all hover:shadow-lg">
  开始游戏
</button>

✅ 简化后（一个类名）

<button className="btn-primary">
  开始游戏
</button>

按钮组件

<button className="btn-primary">开始游戏</button>
<button className="btn-secondary">加入房间</button>

可用的简化类名

按钮

  • .btn-primary - 主按钮
  • .btn-secondary - 次要按钮

卡片

  • .card - 基础卡片
  • .card-hover - 可悬停卡片
  • .card-title - 卡片标题

输入框

  • .input - 基础输入框

玩家颜色

  • .player-1 ~ .player-8

地图

  • .map-cell - 地图单元格

布局

  • .section - 区块容器
  • .section-title - 区块标题

为什么要简化？

✅ 代码更简洁

从 10+ 个类名缩减到 1-2 个，代码可读性大大提升

✅ 一致性更好

团队成员都使用相同的组件类，UI 风格统一

✅ 维护更容易

修改设计时只需在 globals.css 中改一处，全项目生效

✅ 灵活性依然保留

可以继续用 Tailwind 原生类进行微调：btn-primary text-lg
